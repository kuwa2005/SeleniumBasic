# SeleniumBasic フォーク — 基本方針メモ

この文書は、このフォークで開発・変更を行うときの**判断の軸**を残すためのメモである。法的文書やコントリビューション規約の代替ではない。

## 目的

1. **現代のツールチェーンでビルドできる**ようにする（コンパイラ・SDK・プロジェクト形式など）。
2. **バグを修正**する。
3. **十年前後のブラウザ前提で成立していた前提**が、**現在のブラウザでは成立しなくなった部分**を、必要に応じて**拡張・更新**する。
4. 元リポジトリ（upstream）の **Issue / 未マージ PR** のうち、**未対応で有用そうなもの**は積極的に**候補として取り上げ、実装**する。  
   - 元 Issue の内容が常に正しいとは限らないため、**再現手順・影響範囲の確認**を挟む。

元リポジトリ: <https://github.com/florentbr/SeleniumBasic>

## ブラウザ範囲（このフォーク）

- **メンテナンス・実装の対象は Google Chrome（chromedriver）のみ**とする。
- **Firefox / Internet Explorer / Microsoft Edge / Opera / PhantomJS** については、**COM の ProgId・型・メソッド署名は維持**するが、**起動ロジックは削除済み**（`Start` / `StartRemotely` は chrome のみ通す。他ブラウザ名は `ChromeOnlyBrowserError`）。
- PhantomJS は SSL 等の理由により**サポート対象外**（スタブのみ）。

## 絶対的必須：下位互換性

- **ランタイムの問題**（特定 OS・ブラウザ版・ドライバの世代・ネットワークやタイミングなど**環境依存の成否**）は、方針決定の議論では**いったん切り離してよい**。  
  ただし後から手を入れる際は、**互換を壊さない形**を優先する。
- **下位互換性には最大限配慮する。** 具体的には次を原則とする。

### API / COM / スクリプトから見た契約

- 既存の **VBA / VBScript / VB.NET** から利用される **メソッド名・引数の意味・戻り値・省略時の既定動作**を、**強い理由がない限り変えない**。
- 新機能が必要な場合は、可能なら **別メソッド・オプション引数・明示フラグ**など、**既存呼び出しを壊さない**形で足す。
- **既定動作（デフォルト）を「正しい方」に変える**変更は、既存スクリプトの意味が変わりうるため、**デフォルトでは行わない**。どうしても必要なら **明示的な新 API / オプション** に寄せる。

### データ・設定・エラー

- 読み書きする **ファイル形式・設定・レジストリ** 等がある場合、**古い形式は必ず読める**ようにする（新形式の導入は読み手を壊さない順序で）。
- 呼び出し側が **エラーコードやメッセージで分岐**している可能性を考え、**既存のエラー体系を無断で改名・統合**しない。

## 実装・運用の目安

- 元リポジトリ Issue の**取り込み候補と推奨実装順**は `UPSTREAM_BACKLOG.md` を参照（随時更新）。
- **chromedriver** の探索順は `README.md`（環境変数・サブフォルダ・PATH・`SetChromeDriverPath`）。
- upstream からの更新は `upstream` リモートで **定期的に fetch** し、必要なら取り込む。
- 元 Issue / PR を手がける場合、自分のフォークでは **「Upstream #番号」へのリンク**と要約を Issue やコミットメッセージに残すと追跡しやすい。
- 変更は **小さな単位のコミット** とし、**互換に触れるか**を意識した説明を付ける。

## コミット・PR・Issue の言語（このフォーク）

- **コミットメッセージ**は原則 **日本語**とする（件名＋必要なら本文で理由・影響を書く）。
- **プルリクエスト**のタイトル・説明・レビューコメントも **日本語**を用いる。
- **このフォーク上の Issue** も **日本語**を基本とする（upstream へのリンクや引用は英語のままでよい）。
